What are the ingredients?

While developing the front-end framework of Theme Engine we had three main goals; to make it flexible, well-documented and easy to maintain.

To achieve these goals we used a set tools and methods with proven track records and comprehensive documentation.

Twitter Bootstrap (Sass)

Designed to make development easier and efficient. Bootstrap is fully responsive to everything from the mobile device to the desktop.

Sass & Compass

The popular CSS-extension language, Sass, powers Compass to allow for code that’s DRY, readable and easy to maintain.

BEM syntax

BEM (block, element, modifier) is a methodology for creating code that’s easy to read, maintain and reuse across themes.

Volusion Storefront API

The Volusion Storefront API offers everything you need to build unique storefront experiences for your theme and beyond.

AngularJS

AngularJS is an open-sourced framework used in the creation of single-page web applications. While it is not required to know AngularJS when developing a new Volusion theme. It is recommened if you plan on extending it's functionaility or developing new components.

Editing your theme

Once you've chosen your base you'll need to download the assets and install a few dependencies. Below are the absolute essentials need to get you started. Refer to our Advanced Guide for more options.

Download your boilerplate

A few options are available when it comes to downloading your boilerplate. The quickest way is to use the official Github app for either Mac or Windows. If you'd like to download the source code directly from Github you can do so via command line with the following command.

If you're planning on contributing, then it's a good idea to fork the repository. You can find instructions for forking a repository at http://help.github.com/fork-a-repo/. After forking the repository, you'll want to create a local copy of your fork. 
            
git clone git://github.com/username/repo.git

Installing dependencies

Here we'll install the dependencies needed to enjoy many of the development features within the Theme Engine.

  • Node.JS
  • npm
  • Ruby

Following the above requirements, you need to install some build tools by running the following commands in your shell:

            
gem install bundler
npm install -g grunt-cli bower

Now you can install the project dependencies:

            
cd theme
npm install

Now when you run grunt serve, it will be running your local copy and showing your changes.

Note: It's a good idea to re-rerun npm install and bower install whenever you pull from the main repository to ensure you have the latest development dependencies.

Directory Structure

Below is a simplified directory of the typical theme file structure:

  • docs - documentation for the project
  • app - contains the source code
    • bower_components - source code and test package dependencies
    • fonts - font files
    • images - images and sprites
    • scripts - JavaScript files
    • styles - Sass files (SCSS syntax with Compass)
    • translations - language translations
    • views - HTML views
  • test - the main unit test folder
    • controllers - tests for the controllers
    • views - tests for the views
    • directives - tests for the directives
    • etc.

Customizing your theme

Here we will discuss common best practices for where to place new code and design assets when customizing your theme. Following these best practices will help keep your themes assets organized and readable.

Adjusting Bootstrap variables

All theme variables can be found located within the folder styes > style-guide. Here you can adjust many of the themes default component styles provided by Bootstrap.

Warning: Do not edit colors within this file. Each color here uses a map-get() to fetch color from the _colors.scss partial.

Extending Bootstrap styles

In some circumstances you will find a need to adjust or extend certain Bootstrap components beyond what is provided in the _theme-vars.scss. To do so, locate the Bootstrap partial within the themes styles directory and add you styles.

Note: Colors within the _theme-vars.scss now reference their color from the _colors.scss file located in the same directory.

Adding new component styles

When adding a new component to your Volusion theme we suggest creating a new Sass partial (_component.scss) and placing it in the themes component folder located at styles > components > th (Be sure to also include it in your main.scss file).

Basic component styling

Below is an example of how styling for a new component should be orginized. Using the nesting features of Sass along with the class naming convention of BEM you can keep code tidy and readable.

Be sure to include the prefix "th-" when adding any new component. This helps future developers distinguish between Bootstrap (no prefix), Theme ("th-") and Volusion ("vn-") specific components. 
              
.th-block {
  &__element {
    width: 50px;
  }
  &--modifier {
    width: 100px;
  }
}

Colors

We highly recommend not placing new colors directly into your style sheets. Instead locate or define the color within _colors.scss. Within the $userDefined map you will see colors defined specifically for the theme to be exposed to the user. Below that you can see where modified versions have been defined. To adjust color variables provided by Bootstrap you can do so within the $bootstrap map. Below is an example of how to call these colors within your css

              
.th-component {
  &__element {
    background-color: map-get($colors, colorName);
  }
  &--modifier {
    background-color: map-get($colors, colorName--modified);
  }
}

Fonts

Fonts can be located within the fonts directory. when adding a new font you will need to follow a few steps in order to get them rendering correctly.

First, place any new font files (.ttf, .eot, .woff, .svg) within the fonts folder. Each file will need to be renamed to reflect the name you will call it within your css. For example, if you plan on calling font-family: Roboto-Italic; you will need to also name the file Roboto-Italic.woff

Note: Each new .svg file added will need to have its id changed to "font".

Second, within the _type.scss partial located within the styles > core folder, you will need to call a the mixin designed to pull in your new font. Below is an example of how this mixin can look within _type.scss file.

              
@include app-font('Roboto-Italic');
