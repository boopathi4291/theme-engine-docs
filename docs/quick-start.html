Not every developer is the same. So why should we serve you the same boilerplate? We all like choices. Enjoy!

Find a starter above that fits both your needs and goals. Seasoned developers and designers might want to start with plain Vanilla, giving them a blank canvas with lots of options. Others may want to begin with established full flavor. We won't judge!

What are the ingredients?

While developing the front-end framework of Theme Engine we had three main goals; to make it flexible, well-documented and easy to maintain.

To achieve these goals we used tools and methods with a proven track record and comprehensive documentation.

Twitter Bootstrap (Sass)

Designed to make development easier and efficient. Bootstrap is fully responsive to everything from the mobile device to the desktop.

Sass & Compass

The popular CSS-extension language, Sass, powers Compass to allow for code that’s DRY, readable and easy to maintain.

BEM syntax

BEM (block, element, modifier) is a methodology for creating code that’s easy to read, maintain and reuse across themes.

Volusion Storefront API

The Volusion Storefront API offers everything you need to build unique storefront experiences for your theme and beyond.

AngularJS

AngularJS is an open-sourced framework used in the creation of single-page web applications. While it is not required to know AngularJS when developing a new Volusion theme. It is recommened if you plan on extending it's functionaility or developing new components.

Editing your theme

Once you've chosen your base you'll need to download the assets and install a few dependencies. Below is a quickstart quide to help you get going in the right direction.

Download your theme boilerplate

A few options are available when it comes to downloading your boilerplate. The easiest way to get going quickly the Github app for either Mac or Windows. If you'd like to download the source directly from Github you can do so via command line with the line below.

If you're planning on contributing, then it's a good idea to fork the repository. You can find instructions for forking a repository at http://help.github.com/fork-a-repo/. After forking the repository, you'll want to create a local copy of your fork. 
            
git clone git://github.com/username/repo.git

Installing dependencies

Here we'll install the dependencies needed to enjoy many of the development features within the Theme Engine.

  • Node.JS
  • npm
  • Ruby

Following the above requirements, you need to install some build tools by running the following commands in your shell:

            
gem install compass
npm install -g grunt-cli bower

Now you can install the project dependencies:

            
cd seed
npm install
bower install

Now when you run grunt serve, it will be running your local copy and showing your changes.

Note: It's a good idea to re-rerun npm install and bower install whenever you pull from the main repository to ensure you have the latest development dependencies.

Directory Structure

Below is a simplified directory of the typical theme file structure:

  • docs - documentation for the project
  • app - contains the source code
    • bower_components - source code and test package dependencies
    • images - images and sprites
    • scripts - JavaScript files
    • styles - Sass files (SCSS syntax with Compass)
    • templates - HTML templates (partial views)
    • translations - language translations
    • views - HTML views
  • test - the main unit test folder
    • controllers - tests for the controllers
    • views - tests for the views
    • directives - tests for the directives
    • etc.

Styling your theme

Knowing where to place your styles and how to orginize them will help you keep you code clean, DRY and maintainable.

BEM and where to place block partials

Using BEM is the best way to achieve this. Below is an example on how to orginize your blocks using BEM and Sass. This block would end up as it's own partial (_block.scss) located within the styles > components > th folder

              
.block {
  &__element {
    width: 50px;
  }
  &--modifier {
    width: 100px;
  }
}

Adjusting Bootstrap variables

A themes variables can be found located within the styes > style-guide _theme-vars.scss file. Here you can adjust many of the themes default Sass variables.

Warning: Do not touch colors within this file. Each color here uses a map-get() to fetch color from the _colors.scss partial.
