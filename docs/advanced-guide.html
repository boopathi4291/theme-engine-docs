Advanced Guide

Welcome

This Getting Started Guide is intended for developers who wish to:

  • Contribute code to the Theme Engine
  • Create their own themes or components

If that sounds like you, then continue reading and let’s get started!

  • You know Javascript.
  • You're comfortable with the command line.
  • You understand unit tests and why they're important.
  • You enjoy creating really cool things with the latest technologies.

If that sounds like you, then continue reading and let’s get started!

Depending upon your comfort level you can either start from a blank slate using our Javascript SDK or choose from one of our pre-built themes and go from there.

Environment Setup

If you choose to use one of our pre-built themes as a starting point, you will need to get your local development environment setup and ready to rock and roll.

GitHub

First, if you don’t already have one, setup a free GitHub profile here.

Next, unless you're a git wizard (a gizard?), you'll want to also install a git client. There are a number to choose from, but the official GitHub ones are actually pretty nice.

For Windows users, navigate here and run the installer.

For Mac users, navigate here and run the installer.

Windows Pro Tip: In the GitHub for Windows client, select Tools >> Options and change your default shell to Git Bash. Select your project, hit the Tools and Options button and then "open a shell here" or just hit the back tick key \` as a shortcut. You now have an excellent Git shell with your Git credentials already stored.

Windows Pro Tip: Add "C:\Users\xxx\AppData\Local\GitHub\PortableGit_xxx\bin" to your PATH user environment variable to gain access to common *nix commands in your Windows Command Prompt without installing msysgit or cygwin.

Node.js

Next, you will need to get Node.js running on your local environment in order to run build tasks, install javascript and css dependencies, and run the development server.

For Windows users, the process is straightforward. Navigate to here and run the Windows Installer.

For Mac users, you’ll need to install Node.js followed by npm. For instructions on this, navigate to here and follow the steps indicated.

Ruby

In order to compile SASS, you will also need to install Ruby. Check out the guides found here for your respective platform.

Build Scripts

This project has several build scripts that help with various parts of development.

grunt test

The primary script to use is grunt test, which does several things:

  • Lints all JavaScript and JSON
  • Runs all tests on Node.js
  • Starts up a development server
  • Runs a subset of tests with Karma

Be sure to run this after making changes and before sending a pull request with your changes.

grunt lint

Runs just the JavaScript and JSON linting on the repository

npm run browserify

Generates build/eslint.js, a version of ESLint for use in the browser

npm run docs

Generates JSDoc documentation and places it into /jsdoc.

npm run profile

This command is used for intensive profiling of ESLint using Chrome Developer Tools. It starts a development server that runs through three profiles:

  • Large - Runs ESLint on JSHint
  • Medium - Runs ESLint on jQuery
  • Small - Runs ESLint on KnockoutJS

Your browser should automatically open to the page in question. When that happens:

  1. Open up developer tools
  2. Click on Profiles

You should start to see profiles for each run show up on the left side. If not, reload the page in the browser. Once all three profiles have completed, they will be available for inspection.

Workflow

Whenever you make changes to the ESLint source files, you'll need to run npm test to rerun the tests. The workflow is:

  1. Make changes
  2. Run npm test to run tests on the command line

You'll have to do this each time you make a change. The tests are run automatically whenever a pull request is received, so make sure to verify your changes work before submitting them.

Source Code

Before you can get started, you'll need to get a copy of the source code. This section explains how to do that and a little about the source code structure.

If you simply want to create a local copy of the source to play with, you can clone the main repository using this command:

git clone git://github.com/volusion-angular/seed.git

If you're planning on contributing, then it's a good idea to fork the repository. You can find instructions for forking a repository at http://help.github.com/fork-a-repo/. After forking the repository, you'll want to create a local copy of your fork.

Start Developing

Following the above requirements, you need to install some build tools by running the following commands in your shell:

gem install bundler
npm install -g grunt-cli bower

Now you can install the project dependencies:

cd seed
npm install

Now when you run grunt serve, it will be running your local copy and showing your changes.

Note: It's a good idea to re-rerun npm install and bower install whenever you pull from the main repository to ensure you have the latest development dependencies.

Directory structure

The directory and file structure is as follows:

  • docs - documentation for the project
  • app - contains the source code
    • bower_components - source code and test package dependencies
    • fonts - fonts used for the theme
    • images - images and sprites
    • scripts - JavaScript files
    • styles - Sass files (SCSS syntax with Compass)
    • translations - language translations
    • views - HTML views
  • test - the main unit test folder
    • spec - tests for the source code
      • controllers - tests for the controllers
      • views - tests for the views
      • directives - tests for the directives
      • etc.

Creating Components

Components are supplimental to a theme and the driving force behind functionality within pages. This section tells you how to get started with component development.

Components represent a unit of a page. Each component is independent. This means they know nothing about each other.

Communication

Components can send messages between each other with $rootScope.$emit:

$rootScope.$emit('add_to_cart', { id: 42 });

Another component can listen to this event with $rootScope.$on:

            
$rootScope.$on('add_to_cart', function(event, item) {
  $cartService.add(item, function(err) {
    if (err) {
      throw err;
    }
    $rootScope.$emit('cart_item_added');
  });
});

See Angularjs API Reference - $rootscope.scope for more information.
