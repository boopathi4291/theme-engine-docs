Advanced Guide

Welcome

This Getting Started Guide is intended for developers who wish to:

  • Contribute code to the Theme Engine
  • Create their own themes or components

If that sounds like you, then continue reading and let’s get started!

  • You know Javascript.
  • You're comfortable with the command line.
  • You understand unit tests and why they're important.
  • You enjoy creating really cool things with the latest technologies.

If that sounds like you, then continue reading and let’s get started!

Depending upon your comfort level you can either start from a blank slate using our Javascript SDK or choose from one of our pre-built themes and go from there.

Get the Code

Advanced users, if you haven't setup your environment yet, refer to the "Setup Your Environment" section in the Quick Start Guide to in order to use one of our starter themes.

Before you can get started, you'll need to get a copy of the source code. This section explains how to do that and a little about the source code structure.

If you simply want to create a local copy of the source to play with, you can clone the main repository using this command:

git clone git://github.com/volusion-angular/seed.git

If you're planning on contributing, then it's a good idea to fork the repository. You can find instructions for forking a repository at http://help.github.com/fork-a-repo/. After forking the repository, you'll want to create a local copy of your fork.

Install Project Dependencies

Open your Git command shell and run the following commands:

cd theme
npm install

Now when you run grunt serve, it will be running your local copy and showing your changes.

Note: It's a good idea to re-rerun npm install and bower install whenever you pull from the main repository to ensure you have the latest development dependencies.

Directory structure

The directory and file structure is as follows:

  • docs - documentation for the project
  • app - contains the source code
    • bower_components - source code and test package dependencies
    • fonts - fonts used for the theme
    • images - images and sprites
    • scripts - JavaScript files
    • styles - Sass files (SCSS syntax with Compass)
    • translations - language translations
    • views - HTML views
  • test - the main unit test folder
    • spec - tests for the source code
      • controllers - tests for the controllers
      • views - tests for the views
      • directives - tests for the directives
      • etc.

Creating Components

Components are supplimental to a theme and the driving force behind functionality within pages. This section tells you how to get started with component development.

Components represent a unit of a page. Each component is independent. This means they know nothing about each other.

Communication

Components can send messages between each other with $rootScope.$emit:

$rootScope.$emit('add_to_cart', { id: 42 });

Another component can listen to this event with $rootScope.$on:

            
$rootScope.$on('add_to_cart', function(event, item) {
  $cartService.add(item, function(err) {
    if (err) {
      throw err;
    }
    $rootScope.$emit('cart_item_added');
  });
});

See Angularjs API Reference - $rootscope.scope for more information.

Layout

A component represents a unit of a page. Each component is independent. This means that they know nothing about each other. To make them communicate, a Publish/Subscribe (Mediator) pattern is used.
