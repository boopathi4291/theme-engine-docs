Theme Folder Structure
A quick overview of where files are.

After you have created a copy of the theme files from GitHub (by clone, branch, fork or other method), you can begin digging into the files and customizing the theme formats.

The following table details a simplified directory of the typical theme folder structure and file locations. The main folders in the theme are:

  • app - Source code for the theme, scripts, fonts, images, styles, and more
  • config - Configuration settings for the theme and environment

Depending on your installed tools and applications, you may have additional folders display in your Volusion theme. When creating your own themes by scratch, or extending one of the Volusion seed themes, you will want to mirror this folder structure.

App Folder

Folder Description
bower_components The source code and test package dependencies.
fonts All font files used by the theme. For each font you add, include all files from a webfont package: embedded open type (.eot), scalable vector graphic (.svg), truetype (.ttf), and web open font format (.woff). Learn more here!
images Images and sprites used in your storefront. We recommend using JPG and PNG file types. The images should be in folders as used in the themes. Learn more here!
scripts

JavaScript files used by the theme, including a custom app (app.js). This folder includes another set of folders:

  • controllers - Provide JavaScript functions bound to a particular scope (connects a View with a Controller). The controller passes data to a view through the scope. The Method theme includes a controller per view to make calls to the Storefront API, providing that data to the view for displaying products, categories, and other content to your storefront. Controllers are one part of the key elements needed for creating custom API interactions with your storefront.
  • decorators - Allows you to make slight changes on third-party and upstream services for use with your modules, while leaving the module intact. Example of decorator actions include caching, debugging/tracing, and throttling. In the Method theme, a decorator is used for wrapping around the translation service.
  • directives - Application special behaviors to perform, matched with DOM elements in a page/code (such as an attribute, element name, comment, or style class). For example, the Method theme includes an easy-zoom.js directive that zooms in and out on an image when an element is used.
  • filters - Customized filter formats for values in the theme. for example, the Method theme includes an seoFriendly.js filter for formatting SEO content and values.
  • services - Customized services
settings Contains the themeSettings.json file. This file provides the default settings available to Merchants that use your theme. For details, see the Method theme tour for info on the file.
styles

Holds all stylesheets as Sass files (SCSS) for your storefront's display. These files control the look and feel of your entire storefront. The main.scss stylesheet imports all styles added to its list to provide a single stylesheet to your storefront. This allows you to breakdown lengthy style formatting into multiple files, in multiple folders as needed, to plan out and update styles.

If adding a new stylesheet, make sure to add an include to this file. Styles are used in the view files for setting section sizes, colors, fonts, and other formatted settings.

For more information, see Storefront Styles.

This folder includes addition subfolders:

  • components - All stylesheets for all web components in your theme. This includes styles for modals, the slider carousel, alerts, breadcrumbs, and more.
  • core - Stylesheets for the fundamental parts of your theme pages, including buttons, forms, and tables.
  • reset - Stylesheets for importing bootstrap styles, such as normalize and print.
  • style-guide - Stylesheets for the basic colors, fonts, and button looks in your theme. You will use the _fonts.scss to add new fonts to your theme, _buttons.scss to control button text, and _type.scss for your H1. One of the most important files is the _theme-vars.scss which sets variables for all aspects of theme settings and content, usable by the variable name across Bootstrap.
  • utilities - Stylesheets for styling utilities. The files in this folder import bootstrap settings for responsive utilities and utilities.
translations For every language your storefronts supports, you will need to add a JSON file with that content in each folder. The folders correspond to the storefront pages supporting translation text.
views

Views act as templates to display data received from a controller. The controllers make calls to the Storefront API and modify data, sending it through a scope, to display in this view. The themes include a view per controller. These templates provide the container settings for desktop, tablet, and mobile responsive layouts.

  • Page Templates: about, article, category, contact, home, i18n (used for translations), product, style-guide, theme-settings.
  • Partials: footer, header, mobile-menu, product-tile, social-sharing

Views are a key element in your theme for displaying custom content on your storefront.

Config Folder

Folder Description
env

Provides configuration JSON files per environment. These files set the localhost, API URL, if translations are enabled or disabled, the URL prefix, and localization settings for region, language, and country.

Example of dev.json:

{
	"config": {
		"LOCALHOST": true,
		"API_URL": "http://www.samplestore.io/api/v1",
		"DISABLE_TRANSLATIONS": true,
		"URL_PREFIX": "",
		"REGION": "",
		"LANG": "",
		"COUNTRY": ""
	}
}
