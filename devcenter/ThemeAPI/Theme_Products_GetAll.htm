Get a Collection of All Products

This GET method requests and receives all possible products in the store. Depending on the size of the store, the amount could be extensive. To refine the returned results, the request can specify specific categories, search terms, facets, minimum and maximum price, accessories, sorting and pagination for a limited set of all products.

You can use this method to provide content for displays custom categories in the storefront navigation, custom sorting by additional product values, and much more through interactions with additional endpoints and your own creativity.

Related Products

This method also supports getting all related products for a product. Related products share the same subcategory as an entered product code. These products could be best bought together or as alternate options, such as other types of running shoes or other hiking accessories for a trip. To GET related products, you can call this endpoint using the category ID: /products?categoryId={id}.

Accessories

You can GET all associated accessories as well. Accessories are associated products in your store that are assigned as Accessories during product creation or editing. For example, you could have a MP3 Player on your store with multiple accessory products including headphones and power cables (wall plug, car, USB computer). To GET accessories, you can call this endpoint passing one or more product codes through the accessoryOf parameter: /products?accessoriesOf={accessoriesOf}.

Request

The URI call structure is:

GET http://{domain}/api/v1/products/?categoryIds=categoryIds&slug=slug&search=search&facets=facets&minPrice=minPrice&maxPrice=maxPrice&accesoriesOf=accessoriesOf&sort=sort&pageNumber=pageNumber&pageSize=pageSize

Parameters

The following parameters can be entered into the request:

Parameter Description Type
categoryIds Numeric ID of the category of associated products to retrieve. Example: 1347. Number
slug String of the product name, which is the SEO optimized URL keywords. Example: fitness-shoe-mens. String
search A searchable String using the first four words separated by spaces. String
facets String for filtering the products using filter category IDs. For example, a String of 1822, 1818, 1829 would return those products that are Orange [1822], Red [1818], Brown [1832]. String
minPrice Sets the minimum price for products. All returned products will have a price equal to or greater than this amount. Example: 25. Number
maxPrice Sets the maximum price for products. All returned products will have a price equal to or less than this amount. Example: 100. Number
accessoriesOf Retrieves the accessory products for the entered product code(s) in this String. String
sort Sets the sort priority using one of the following: relevance, lowest price, highest price, newest, oldest, or popularity. Example: relevance. String
pageNumber Numeric page number of the returned products to retrieve. Example: 1. Number
pageSize Numeric amount of products to retrieve for the page. Example: 10. Number

Data

The response includes a section of JSON detailing the pagination of the product content for displaying and navigating the collection. A data collection follows with detailed information per product.

The following cursor data is returned to set the pagination of response product data:

Attribute Description Type
total Numeric amount of all products returned matching the request. Number
totalPages Numeric amount of total pages for pagination of the returned products. The total amount is divided across the total amount of pages. Number

The following data is returned in a collection, one data element per product for the request:

Attribute Description Type
id The generated ID for the product. This is generated automatically by the Volusion system. Example: 37349 String
code The product code you entered at product creation. This is an alphanumeric entry that can include some special characters. Example: ah-chest String
name The product name entered at product creation that describes the product. The name is alphanumeric and supports some special characters. This is a required setting and will always return content. Example: 3-Drawer Chest String
price

The price includes a number of available options for setting the shopper payment for the product:

  • regularPrice - This is the price of the product entered at product creation. The price is required and always returns a value. Example: 300
  • listPrice - This is the recommended retail price for the product.
  • recurringPrice - Provides information for any recurring price options for the product, such as with a subscription product. This price includes a starting price, normally charged price, and data on the length of term for charging each price.
    • price - The normally charged price for the recurring product that begins charging after the initial starting price completes its term.
    • howOften - Sets the recurring charge frequency in months.
    • duration - Sets the complete duration of the recurring price in months. This terms starts after the starting price term ends.
    • startDuration - Sets the complete duration of the starting price recurring charge. After this term, the normal duration begins.
    • startPrice - Sets the initial price for the recurring charges, only charged during the starting duration.

The shipping information for the product, typically returning the weight. Additional shipping information may display or be included for the product as viewed through the Admin.

 Number
descriptions

Every product can include multiple entries for descriptions that display throughout the storefront, the mobile app, through search engines, and for SEO. This information includes the following options:

  • features - Provides specific product feature information that displays in a bulleted list on a Features tab for the product. This entry supports plain text and HTML formatting.
  • extendedInfo - Provides any additional information for the product in an Extended Information tab for the product. This entry supports plain text and HTML formatting.
  • techSpecs - Provides specific technical information that displays in a bulleted list on a Technical Specs tab for the product. This entry supports plain text and HTML formatting.
  • photoCaptionText - Provides a line of text to display below a product image, as a caption for that image.
  • short - Provides a short description for the product, used on category and search results pages performed within the storefront. It is also used when using the email social option for a product. The content supports plain text, HTML formatting, and images.
  • detail - Provides a full description of the product, used by search engines. The content supports plain text, HTML formatting, and images.
 String
seoFriendlyProductTitle The data provides the SEO friendly product name. This option is extremely helpful for searches performed by search engines. Example: 3-Drawer-Chest String
photoAltText This option sets the alternate text to display for a product image. String
isGiftWrapAvailable This setting indicates if the product can be gift wrapped. If this setting is true, the cost of the gift wrap displays for giftwrap price. True/False String
isTaxable This setting indicates if the product can be taxed, or is non-taxed. If false, taxes are not applied to this product during purchase. Non-taxed products vary state-to-state. True/False String
quantity This setting indicates the minimum quantity allowed to purchase. Number
images

This setting returns an array of values for every image available for the product. The digital size of the images are important to consider not only for quality and zooming in, but bandwidth usage (a charged amount per Volusion account). You may want to use a different sized image for initial page loads in your theme to balance bandwidth usage.

Each image includes the following:

  • id - Sets the generated ID for the photo when uploaded. The first image is always 1.
  • medium - Sets the URL to the medium size image file.
  • large - Sets the URL to the large size image file. This file allows for zooming in for details.
  • small - Sets the URL to the small size image file. This image is typically used for thumbnails.
  • type - Sets the image type. Example: photo.

seo

This data provides a series of SEO values:

  • metaTagTitle - Sets the product name for the menu bar of the product page. If NULL, the storefront and search engines default to the product name.
  • metaTagKeywords - Sets the keywords used by search engines to locate products.
  • metaTagDescription - Sets the product description used by search engines when displaying the product in results.
  • metaTagsOverride - Sets advanced code that overrides all existing HTML meta tag information. This option is for advanced users.
 String
availability

This setting provides information regarding the current availability of the product and allowance of backorders.

  • description - Provides a list of descriptive options for the availability, including custom text content.
  • allowBackOrders - Indicates if backorders are allowed for the product. If true, the product still displays on the storefront and accepts more orders. The stock status will display a negative number for the product if tracked.
  • stock - Setting that indicates the stock status of the currently available amount of the product in-stock. If this amount is NULL, the stock of the product is not tracked and always considered "in-stock".
