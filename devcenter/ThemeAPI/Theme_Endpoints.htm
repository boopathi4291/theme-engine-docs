REST API Endpoints

The following is a detailed list of all REST API endpoints in the Storefront API. When entering a request, you will follow a specific format. For details, learn more about the request structure content and response data. Each listed API includes a link to further information, sample JSON requests and responses, and in-depth information for the implementation of your code.

The following are the currently available API endpoints:

Endpoint Description
products Provides access to actions for all products in the store including getting specific products, all product, related products, product accessories, and reviews. Products are goods, services and downloadable items sold through the storefront that earn revenue for the merchant. These products include physical items sold and shipped from a warehouse and downloadable song and book files.
categories Provides a hierarchy of structure into groups of categories and subcategories for products to display on the storefront. For example, you may have a category of shoes with subcategories based on mens, womens and kids with further categorization on the type of shoes or usage from dressing, casual and sports.
articles Provides the content pages for your storefront.
navigation Sets the navigation menus for shoppers to browse products and content on your storefront. Volusion supports up to eight navigation menus on the storefront. The number of menu signifies a section of the storefront menu to access or modify.
config Provides information on the storefront site itself, all configuration through various options in the Admin Settings section, company information, site specific details, and payment settings.
components Provides a collection of components. These options enhance the storefront pages, such as the slider for images and linked content.
carts Provides a shopping cart object for a shopper, including the items in the cart and checkout details (discounts, shipping and billing address, shipping methods, payment profile, miscellaneous and custom attributes, and customer ID). The cart ID may not be required as this information is passed through via a cookie.
countries Provides your enabled countries and states for your storefront.
users Provides options for viewing, creating and updating a user account for your storefront, including the account information, stored addresses, payment profiles and user preferences.
login (states to use legacy endpoint...what is that?)
orders Creates an order. If using the endpoint for guest checkout, a specific set of actions occur using the users endpoint prior to creating the order. Returning customers do not include the users endpoint actions.
