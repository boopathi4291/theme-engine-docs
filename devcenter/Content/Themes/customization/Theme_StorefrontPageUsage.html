Storefront Theme Page Usage

Adding a responsive theme to your store changes specific pages to become "theme-specific". This means, specific pages load the customized responsive theme layouts, formats, styles, and content over the legacy templates you previously used for your store. Each customized responsive theme may affect these pages differently, depending on the designed content, styles, and web components.

For all pages in your added responsive theme, the following occurs:

  • As shoppers access pages, they may be redirected to the new responsive theme page or a legacy page.
  • All pages will use the customized image, CSS style code, fonts, and apps added to your theme.
  • The Theme Engine determines served up pages and content based on entered settings.
  • The Storefront API provides data responses to entered requests in the responsive theme code.

The following sections detail what those pages are, what they support, and how they are handled.

New Responsive Theme Pages

The responsive theme provides extensibility and customization for specific pages, depending on the supported theme-specific features. Each Volusion theme seed project may include different options and formats, best used for specific business verticals or current storefront content.

Not all theme seed project provide all of these listed features immediately after you create a copy of the code. This gives you the opportunity to use the Angular framework, developer code, and designer skills to create your own extended features.

For the examples in this guide, we are looking at the Method theme seed project. This project does not support the following options without editing.

When a user in a responsive theme clicks Add To Cart, the storefront detects the screen size and system loading the page, such as a desktop browser, mobile device, or smart phone. If a smart phone is detected, mobile checkout (/checkout.asp) is used for completing purchases if your site has mobile enabled. Otherwise, depending on the screen size, the shopper is sent to /shoppingcart.asp then to /one-page-checkout.asp during purchase completion.

Legacy and Responsive Storefront Pages

Specific legacy theme storefront pages will redirect to responsive theme pages as supported. These pages automatically redirect according to the following mapping when a responsive theme is added and activated. Numerous other legacy pages will continue to use those legacy templates, with the customized styles and images you added with the installed theme.

As the project continues, the redirection and availability of these pages within the project seed themes, or extendable and customizable by your custom code, may change. This information will be updated as quickly as possible to match the latest theme and API support.

Theme Settings Editor in Admin

The following features will become "theme-specific" with direct editing options within the Admin, accessible in future additions through the Admin. Until the editor and settings are available, you can plan and code towards the new Admin feature.

Feature Legacy Theme Responsive Theme Engine
Display Settings Category grid options are used.

The theme designer can optionally expose different grid styles. The extent of designs to be supported and customizable through the Admin will be part of the Storefront Editor.
Language Article Spot Keys are used. This is a global set of text labels specific to the legacy Volusion storefront design.

All translation and language support, such as labels or small content blocks on a theme, will come with each Theme. You can add translation text in JSON using the framework. The extent of site translation to be supported and customizable through the Admin will be part of the Storefront Editor.
Storefront Editor Ability to edit your home page content block and text/html in your template using the Storefront Edit toolbar.

The Storefront Editor toolbar will be available in future releases. This editor will access a limited set of always available settings expected per theme, such as language support and sets of display and style options. Full details for this editor will be available as the project continues.
