Custom Content

Enter custom text and translations for the theme.

Table of Contents

  1. Folder Location
  2. File Information
  3. Translations

Every theme provides support for custom text per page and multiple translations of your storefront. You can create as many language files as you need per language through the themes.

The following information details how to modify custom text per page and add translation content for your theme using the Method theme as an example.

Volusion uses the Angular.js translation feature for pulling and displaying text based on a selected language.

Folder Location

All custom text and translated content is saved per file, per page, in the {your repo}\method\app\translations folder. Every page and section of the theme is broken down into folders, such as:

Folder Description
index The main landing page
category All category support
about Displayed translated content for your About Us page
contact Translated contact information. You could also use this as information per international location.
home Specific translations per the home page
product Translations for the product page(s)
style-guide  
theme-settings  

File Information

Every file should be named with the two-letter country code and an extension of .json. For US English, default files will include en.json files per folder. You can create a copy of these folders, rename them, and edit the content with your translations. As that language is loaded, the framework and system will load that selected language's file for those pieces of content. For example, you could create a es.json file per every folder for Spanish.

For a list of all supported locales, check the Angular list for ngLocale.

Translations

Translation loading of your storefront is controlled by a set of variables for your environment. You can enable translations then set the supported languages to support additional locales in the storefront.

The environment config files located at {your repo}\method\config\env set if translations are enabled or disabled. By default, translations are disabled for the development (dev.json) and production (prod.json) environments.

To enable translations:

  1. Open the folder location: {your repo}\method\config\env.
  2. Open each environment JSON file and locate the line: "DISABLE_TRANSLATIONS": true,
  3. Change true to false and save.
