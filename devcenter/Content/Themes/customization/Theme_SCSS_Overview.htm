Storefront Style Overview

A general overview of styles with links for more info.

Styles are a powerful, immediate way to add customization and branding flair to your storefront. Styles typically have been a series of formatting selections, documented as code within a CSS file (cascading style sheet) that enforced a single view site-wide. Through the CSS file(s), you could design and maintain the exact look and feel you want for all text content, display controls such as tables and menus, and assets such as images and videos all from a single controlled place.

Sass Styles and BEM Formatting

Moving beyond simple CSS, Volusion has embraced Sass styles for all stylesheets. These files have a .scss file partial. This format is compatible with all versions of CSS, giving you the option of taking current CSS stylesheets and moving them into SCSS. To help using Sass, you may have installed the Compass framework as a Ruby gem.

Need more information?

  • Sass guide and documentation.
  • Compass documentation, the Ruby gem you installed to use Sass.

Style code in SCSS files follows the definition patterns of BEM (block, element, modifier). Each line of code follows a specific naming convention:

.block {} - Represents the higher level of an abstraction or component.
.block__element {} - Represents a descendent of .block that helps form .block as a whole.
.block--modifier {} - Represents a different state or version of .block.

Volusion Theme SCSS

The following style and file information is based on the Method theme seed project. All styles are located in the {your repo}\method\app\styles\ folder.

The main.scss stylesheet combines all imported stylesheets to provide a single stylesheet to your storefront. If adding a new stylesheet, make sure to add an include to this file using just the file name without the extension, in a format of @import 'stylesheetname' such as @import 'components/th/header' for the theme header. Styles are used in the view files for setting section sizes, colors, fonts, and other formatted settings.

The styles folder includes the following subfolders and files:

Folder/File Description
main.scss This file is the main stylesheet for the entire theme using Sass. Within the folder are @include links to extended stylesheets located in subdirectories. This file also contains @import links for imported CSS files from CDN locations. When modifying your styles, you should create new CSS files or add content to the partial Sass sheets.
_th-import.scss Partial Sass stylesheet that imports all partials from the th folder.
overrides.css Any added and changed styles added by merchants using your theme will be saved to this stylesheet file. These styles are typically entered by the merchant/designers activating the theme to their storefront. Overriding styles can be entered directly to this file or using the theme settings through the Admin (or theme settings page).
base folder Includes all partial Sass stylesheets for bootstrap, fonts, and helpers.
  • _bootstrap-style.scss - Review and modify the bootstrap styles for the theme throug this extension.
  • _bootstrap-variables.scss - Contains all variables used by the theme to associate and use styles in the theme. For example, you can add new fonts to variables for all font families and styles that use fonts. All used styles in the theme templates and pages automatically inherit the new fonts without having to touch the page content code.
  • _fonts.scss - Use this file to add or remove fonts to the theme. For details, see Adding Fonts.
  • _helpers.scss - Contains all mixin files and silent classes for the theme. For details, see Supporting Files.
th folder Contains all partial Sass stylesheets for all aspects of the theme including content per template page, mobile content, faceted search, and more! Each file is named to describe the template page it formats for ease of locating and updating those styles. Remember, check the _bootstrap-variables.scss and _bootstrap-style.scss files to update styles and settings at a deeper level (like fonts, links, etc).

Developing Styles

If new to developing styles, you may want to review the documentation for Sass and Compass. Sass provides a set of modules and functions for designing. Compass augments these options, providing automation and extended design capabilities.

  • When you add any new style partial files, always add an @import for the file in the main.scss file located in {your repo}\method\app\styles.
  • All theme variables are located in the _theme-vars.scss file, located in the {your repo}\method\app\styles\style-guide folder. You can adjust and the theme default component styles in these files. For example, when adding fonts, you can add them to this file.
  • When adding a new component to your Volusion theme, you should create a new Sass partial called _component.scss in the {your repo}\method\app\styles\components\th folder.

If using Compass, the following functions and information may be helpful in developing Sass:

  • Helper functions - Augmented functions for colors, constants, fonts, images, sprites, and more.
  • Typography module - Deeper control using mixins for common text styling patterns in links, text, and lists.
  • Layout - Information on grid backgrounds, sticky footers, and stretching.

Adjusting Current Styles

When adjusting the style guide for the entire theme, you want to change settings in _theme-vars.scss and _variables.scss located in the {your repo}\method\app\styles folder.

To learn more about stylesheets, review the Storefront Styles information.

Extending Styles

If you need to extend components for the Style Guide beyond the styles in the variable SCSS files, you can extend the _theme.scss file located in the {your repo}\method\app\styles\style-guide folder.

To learn more about stylesheets, review the Storefront Styles information.

Adding New Component Styles

When you add a new component to your theme, you should create a new Sass partial file called _component.scss and save it into the {your repo}\method\app\styles\component\th folder. You also need to add an @import for the file in the main.scss file located in {your repo}\method\app\styles.

The following is an example for styling a new component and how to organize the code. Using the nesting features of Sass along with the class naming convention of BEM, you can keep code tidy and readable. Make sure to include the prefix th- when adding any new component. the prefix is used to inform developers what kind of component was added: Bootstrap (no prefix), Theme (th-) and Volusion (vn-) components.

.th-block {
	&__element {
		width: 50px;
	}
	&--modifier {
		width: 100px;
	}
}

To learn more about stylesheets, review the Storefront Styles information.

Customizing Colors

As a best practice, you should only modify or add new colors to the variables files: _theme-vars.scss and _variables.scss located in the {your repo}\method\app\styles folder. If none of the colors provided are the exact ones you want, you may want to use the Sass and Compass color functions to adjust Bootstrap variables in your stylesheets.

.th-link-list {
	&__link {
		color: $link-color;
	}
	&--hover {
		color: darken($link-color, 10%);
	}
}

Compiling Styles

This command checks your style files to determine if any changes have occurred and compiles accordingly.

compass watch

When entered, Compass will poll for changes. If saved changes are found in your styles, the command automatically compiles your Sass to CSS.
