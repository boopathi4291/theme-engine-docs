Get Components

This GET method retrieves a collection of all components available and in-use on the storefront. These objects are web components for use in sites to generate custom HTML and CSS interactions and design on a site. Common examples of components in the Volusion Storefront include a slider for image and content across the top of a web page and an area for displaying shopper reviews.

The response returns a collection of data for every component in the system. Due to the amount of components available, the response may be lengthy.

The following information guides you through the request parameters and the response JSON.

Request

The URI call structure is:

GET http://{domain}/api/v1/components

Parameters

This request does not require parameters. It gets all available components in the storefront.

Response

The response JSON provides a status code 200 and data for the components in the storefront. Each type of component will include a collection of instances, each differing on usage and content in the storefront.

Data

The response includes JSON data for each component in the storefront. Some components may return a collection of data, depending on the type of component.

The following data response is for the reviews component:

Attribute Description Type
settings

This information sets the configuration options for displaying shopper reviews.

  • showLocations - Indicates if the entered geographical location displays for the review entry.
  • showSummary - Indicates if a summary displays for the shopper review body content.
 String

The following data response is for the slider component, which can return a collection of slider components:

Attribute Description Type
slides

The nested attributes for the images, text content, and URL link for each slide's displayed content. This content is repeated per slide on the slider:

  • imageURL - The full URL location of the image to display in the slider.
  • headline - A line of text that acts as a title, caption, or eye-catching notice. It displays across the slider image to gain attention and call to action.
  • subHeadline - A secondary line of text providing further description for the slider content.
  • linksTo - A directory link to a specific category or product in the storefront to associate to the slide. For example: product1/p/123 and category1/c/123
  • htmlContent - The URL link and displayed link text to click for viewing additional content for the slide. This setting is optional per slide.
 String
settings

The settings content indicates if arrows or dots are used in the movement and display of the slides:

  • enableArrows - Arrows images are displayed and track movement slide-to-slide in the component. These are clickable elements moving the viewer forward or backward through the slides, one at a time.
  • enableDots - Dot images are displayed and track movement slide-to-slide in the component. These are clickable elements returning the viewer to that slide in the list.
 True/False String
