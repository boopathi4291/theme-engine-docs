Get Navigation Links

This GET method requests a full list of all links for a navigation area of the storefront. The response provides a list of all categories with subcategories linked to the input navigation area by numbered location. The storefront supports up to eight navigation menus that display across the top of side depending on the design of your theme. Depending on the depth of subcategories in your category hierarchy, the response may be lengthy.

Request

The URI call structure is:

GET http://{domain}/api/v1/navs/id?levels=levels

Parameters

The following parameter can be entered into the request:

Parameter Description Type
id Numeric ID for the specific navigation menu area. The options are one through eight for displaying the possible links. This endpoint cannot be used for locating all links for additional navigation areas and categories such as home page (H), search results (S), photo gallery (G), the home page super deals (W)or the home page top sellers (T). Number
levels Depth of categories/subcategories to retrieve, up to 3 levels. Example: 2. Number

Response

The response includes a response code and data for the all links available in the passed numbered navigation area. These links are associated categories and subcategories, displayed according to the design and settings of your storefront navigation. Depending on the hierarchy of your categories and subcategories, this response may be lengthy.

Status Code 200

Data

The response includes a data collection with detailed information per category and associated subcategories all linked and displayed in the entered navigation area. Due to the parent-child relationship of categories, the data may return multiple nested subcategories with further nested subcategories.

The following data is returned for the articles:

Attribute Description Type
id The generated ID for the category when created. This is generated automatically by the Volusion system. Example: 1814 String
name The category name entered at category creation. The name is alphanumeric and supports some special characters. This is a required setting and always returns content. Example: Apparel. String
alternateUrl This URL generates from the entered Category URL Text for the SEO options for a category. A dependency for this value is required through the Marketing > SEO > Search Engine Friendly URLs section of the Admin. With this activated and content entered, a new alternate, SEO friendly category displays. If NULL, the URL may default to the category ID.

String
subCategories

This is a collection of all subcategories to this category. Any category with the Subcategory of... entered with a parent category ID will be considered a subcategory of that parent category and display in that collection. The collection includes the following information:

  • id - The generated ID of the category.
  • name - The entered name for the category.
  • alternateUrl - As the alternateURL above, it is a generated URL using SEO options and entered text to create an easily read and used category URL. Otherwise, it defaults to the category ID.
  • subcategories - A collection of categories with this category listed as the parent category ID.
  • imgUrl - Sets the URL location of the category image to use and display. If no image is included for the category, the setting defaults to a placeholder image URL.
  • description - Fully defined description of the category.
  • seo - This data provides a series of SEO values:
    • metaTagTitle - Sets an SEO friendly name for the category. If NULL, the data defaults to the category name.
    • metaTagKeywords - Sets all keywords used by search engines to locate and return this category in search results. Keywords are separated by commas. This is an unlimited String.
    • metaTagDescriptions - Sets the category description used in search engine results. This is a 255 character String.
    • metaTagsOverride - Sets advanced code that overrides all existing HTML meta tag information. This option is for advanced users.
  • slug- Provides the SEO friendly name for the category. A category may have a long name or contain numerous special characters. This content may not adhere to SEO usage. The name is used when generating the URL.
  
imageUrl This text sets the URL path for the category image. If no image is included for the category, the setting defaults to a placeholder image URL. Example: /v/vspfiles/templates/99999/images/nophoto-0.gif  
description This option sets the description displayed for the category, on the category page when loaded in the storefront. The content can be deeply detailed, supporting plain text and HTML code.  
seo

This data sets the SEO friendly content for the category, including the following:

  • metaTagTitle - Sets an SEO friendly name for the category. If NULL, the data defaults to the category name.
  • metaTagKeywords - Sets all keywords used by search engines to locate and return this category in search results. Keywords are separated by commas. This is an unlimited String.
  • metaTagDescriptions - Sets the category description used in search engine results. This is a 255 character String.
  • metaTagsOverride - Sets advanced code that overrides all existing HTML meta tag information. This option is for advanced users.
 String
slug

This attribute provides the SEO friendly name for the category. A category may have a long name or contain numerous special characters. This content may not adhere to SEO usage. The name is used when generating the URL.
