Get Countries

This GET method requests a full list of all supported countries and their states or provinces. These values are used throughout the storefront checkout processes for any address entered and used by the system. The response includes a country with a collection of states or provinces for that country. If multiple countries are supported, the response will include nested attributes for the country object and its collection.

Request

The URI call structure is:

GET http://{domain}/api/v1/countries

Parameters

This request does not require parameters. It gets all country data supported by the storefront.

Response

The response includes a response code and data for the all supported countries. Each country includes a collection of states or provinces.

Data

The response includes data for the supported countries and a collection of states/provinces.

The following data is returned for the countries:

Attribute Description Type
name The name of the country. Example: United States. String
Id The generated ID for the country when added to the system in the Admin. String
StateLabel The label for the state or province. For the United States, this label would be State. For Canada, this label would be Province.

String
States

A collection of states/provinces for the specified country. Each state includes the following attributes:

  • Id - The ID for the state/province for use within the storefront, generated by the system.
  • Code - The accepted acronym or initials for the state/province. This code is used in addresses throughout the storefront and Admin. It is also an accepted postal code for the state/province used throughout the world for specific locations.
  • Name - The full name for the state/province.
