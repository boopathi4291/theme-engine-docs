Image Spriting

Image spriting is an emerging way for packaging a number of images, called sprites, into one large image as a sprite sheet. With one request, a site can receive all of the images for a page rather than numerous calls for all images, requested one at a time. Sprite maps allow you to use background positioning to selectively display an image from the map into a specific section or area of your page.

Sprites and sprite sheets can be used for icons, buttons, logos, and other images that are small, compact, and sharing a similar color scheme. These types of images are perfect for sprite maps as they all share the same pixel dimensions for size, giving you multiple "tiles" to select and use from the map into your design.

Great Idea? Create a sprite map for your social icons! This gives you a powerful and easy way to style your icons to look uniform and clean in layouts.

You can use Sass to create sprites manually or Sass with Compass to generate them all!

Creating Images

When creating and using images with Compass to build sprite maps, you should do the following:

  • Create images with the same dimensions for your icon/button groupings. If images are all variable in size, this can cause issues with the maps and style code.
  • Save all images in a .png format. This format also allows for transparent backgrounds, letting the background colors/images display through from your pages.
  • Make sure to save your .png files for maps in the images folder, preferably grouping images intended for a sprite map. Compass uses .png files as a sprite image folder during an import directive. It will automatically assume that folder of images needs to be converted into a sprite sheet.
  • When using a Volusion theme seed project, you may want to follow the same image folder groupings: images\sprites\{folder name}.
Sass Sprites

You can create any size of image to use as a sprite, as a jpg, svg, or png, for use in your theme. Maps give you the ability to add multiple images all matching in dimensions into a single image for use within SCSS files to insert into your page templates.

If you want to automatically generate sprite maps and CSS code, consider using Compass!

The following image is a sprite map image of common formatting icons for text, images, and attachments. Each icon image is could be 24px by 24px, which becomes the size of each tile. This image contains icons available as buttons for your site, 8 tiles across and two tiles down.

Each style created for the image would designate the icon or button by a number of tiles across and down. The following SCSS code provides the overall sizing for the sprite map and style associated with the media icon. This image could be used when embedding a video on click:

$icon-width: 24px;
$icon-height: 24px;
$icons: image-url('toolbar.png');

.media-icon {
	background-image: $icons;
	background-position: -($icon-width * 5) -($icon-width * 1);
	width: $icon-width;
	height: $icon-height;
}

This code would load an icon image located five tiles over and one tile down, 24px per tile. That image is highlighted in the sprite map below:

When the media-icon style is used, that icon displays!

Consider Compass

While this format for creating sprites is helpful, the overhead for maintaining and adding images can take a great deal of time and require image editing. You would then need to add styles for the new locations of icons in the map, and edit any existing styles if icon locations changed.

If you would rather generate a sprite map and associated styles, you may want to use Compass.

Compass Sprites

Compass sprites build entire sprite maps for you as you save image files into your theme. All images for sprites should be saved to the following location in your theme: {your repo}\method\app\images\sprites. The following information provides insights and image recommendations using Volusion themes.

For the Method theme, sprites are saved in three folders: icons (for buttons), social (for social media sharing), and stars (for reviewers and ratings).

In using Compass, you have a few options for creating sprite maps with directives and mixins. These are just a couple quick ways to generate sprite maps and styles:

  • @import Automatic Sprite Maps - With images saved to a folder, you can use the @import directive to automatically generate sprite maps!
  • @import with Class Names - This option automatically generates the sprite image and allows you to have control over the generated class names.

For further information and examples of everything Compass supports, you can review all possible options from the Compass Spriting documentation and additional examples.

@Import Automatic Option

The import option is the automatic, fastest way to generate a sprite map. This method will automatically create a sprite map with a name plus hash code. The generated styles use the folder and image file name with a background-position based on the order they are added to the sprite.

For example, Method theme has a social folder of sprite images for all social media sites. Each image is the same 27x27px size, 72 dpi.

The following code creates a sprite map using all .png images located in the social folder:

@import "images/sprites/social/*.png";

Using this line of code then generates style classes using that sprite map and the all-{folder name}-sprites mixin:

@import all-social-sprites;

With the sprite map and the all-{folder name}-sprites mixin, CSS code generates a style per image (using the file and folder names):

.social-sprite, .social-facebook, .social-google, .social-instagram {
	background-image: url('../images/social-s1f1c6cbfd0.png');
	background-repeat: no-repeat;
}

.social-facebook {
	background-position: 0 0;
}

.social-google {
	background-position: 0 -27px;
}

.social-instagram {
	background-position: 0 -54px;
}

@import with Class Names

You may want to create style class names that do not use the same name as the folder and file. In this case, you can declare the class name in code, using a different mixin.

For example, the following code creates a sprite map and generates CSS classes using the @include mixin:

@import "images/sprites/social/*.png";
	
.facebook-icon { @include social-sprite(facebook); }
.google-icon { @include social-sprite(google); }
.instagram-icon { @include social-sprite(instagram); }

The generated CSS uses those style names you specified.
