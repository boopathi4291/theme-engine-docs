Response Format

When the API receives your request, a response is returned as a response envelope. Depending on a success or failure, the response envelope may include the returned object data, pagination information, warnings, errors, or exceptions. Responses may also include a status code with an additional message regarding errors or exceptions.

On a success (typically a 200), only the data object is returned. A success of 204 does not return a data object. Each JSON sample for the endpoints provides samples of this response.

The response is returned as a JSON Envelope in the following format:

{
	"data": {},
	"cursor": {}
	"facets": [],
	"categories": [],
	"warnings": [],
	"errors": [],
	"exceptions": []
}

 

The following is a sample response of a successful request:

{
	"data": {
		"id": 12663,
		"firstName": "John",
		"lastName": "Smith",
		"companyName": "Acme, Co",
		"address1": "123 Maple Street",
		"address2": "Suite 1008",
		"city": "Austin",
		"state": "TX",
		"postalCode": "787333",
		"country": "United States",
		"phoneNumber": "123-123-1234",
		"residential": true
	},
	"cursor": {}
	"facets": [],
	"categories": [],
	"warnings": [],
	"errors": [],
	"exceptions": []
}

 

The following table defines the code included in the response envelope.

Code Description
data On success, contains the resource object per the submitted request. This object is only returned on success. If an error or exception occurs, this object is empty. For collections, the response will always indicate success with an empty data array.
cursor

Contains the pagination info for large result sets. The total details the entire amount returned. The totalPages details how many pages these results will be displayed across.

"cursor": {
	"total": 100,
	"totalPages": 5}
facets

Contains the faceted search data used for filtering large result sets. You can set each facet to a specific property type with associated properties. The options use filter categories you have created and defined within the Admin. 

"facets": [
	{
		"id": 1817,
		"title": "Color",
		"properties": [
			{
			"id": 1822,
			"name": "Orange",
			"count": 32
			},
			{
			"id": 1832,
			"name": "Brown",
			"count": 4
			},
			{
			"id": 1825,
			"name": "Black",
			"count": 76
			}
		]
	},
	{
		"id": 1817,
		"title": "Material",
		"properties": [
			{
			"id": 1827,
			"name": "Cotton",
			"count": 21
			},
			{
			"id": 1829,
			"name": "Wood",
			"count": 44
			},
			{
			"id": 1830,
			"name": "Glass",
			"count": 7
			}
		]
	}
]
categories

Contains the categories available for filtering large result sets of data. The accepted content includes main and subcategories.

"categories": [
	{
		"id": 1477,
		"title": "Home Decor",
		"properties": [
			{
			"id": 1516,
			"name": "Furniture",
			"count": 32
			},
			{
			"id": 1334,
			"name": "Art",
			"count": 4
			}
		]
	},
	{
		"id": 1817,
		"title": "Books",
		"properties": [
			{
			"id": 5572,
			"name": "Fiction",
			"count": 9
			},
			{
			"id": 5531,
			"name": "Cooking",
			"count": 12
			}                
		]
	}
]
warnings

Contains information messages regarding data dependencies that trigger warnings. These messages are returned for a success or failure.

"warnings": [
	{
	"message": "An item in your cart has been removed because the item is no longer in stock."
	}
]
errors

Contains validation errors when a request fails. The error provides information for locating the issue and updating the data request. On a success, this data is empty. For example, you may receive an error if a field is required or contain a specific amount of characters or digits. 

"errors": [
	{
	"message": "must be either 6 or 10 digits",
	"relatedToField": "postalCode"
	},
	{
	"message": "required",
	"relatedToField": "country"
	}
]
exceptions Contains exceptions that are thrown when a request fails due to an invalid request or code error. The exception provides information for locating the issue and updating the data request. On a success, this data is empty. For example, you may receive an exception when attempting to update a resource that does not exist.
"exceptions": [
	{
	"code": 404
	"message": "shippingAddress resource 983 does not exist"
	},
	{
	"code": 500
	"message": "The address service failed to perform your requested action"
	}
]
