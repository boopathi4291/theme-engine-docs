Add a Product to a Cart

To add a product to the shopper's cart, you use a POSTmethod. You can add a product using the product code, quantity being purchased, and any configured options available and requested for that product. The endpoint receives the cart ID from the cookie for the shopper, not requiring the cart ID for the method. The response only provides a 201 status code on success.

You can also use theGET method to check the content of a cart for successful add to cart methods.

When passing the entire cart in a POST request, thetotals attribute is the only read-only attribute validated. All other read-only attributes are ignored.

Request

The URI call structure is:

POST http://volusion.apiary-mock.com/api/v1/carts

The header is: 

vMethod: POST

Parameters

The cart ID is not required for the request as a cookie provides any required cart identification. The following parameters can be entered into the request:

Parameter Description Type
code Required. Alphanumeric code entered when creating a product for the storefront. String
qty Required. The numeric amount of the product code to add to the current cart. Number
options

Required. Nested attributes of options selected for the product. These options may include custom text for adding to the product as part of order fulfillment. A product can include multiple options such as size, color, custom text, and material choices.

  • id - Required. The numeric ID for the option available for the product and added to the cart update.
  • optionText - Optional. Text entered with specific punctuation for adding to the product, such as etching, embroidery, and laser printing.

Number

String

Response

The response includes only the status code for success or failure. To verify the product has been added, you can use a GET method on the cart.

Status Code 201
