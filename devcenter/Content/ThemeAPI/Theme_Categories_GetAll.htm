Get a Collection of All Categories

This GET method requests all categories for the storefront. In turn, the response contains a collection for every category-to-subcategory-to-subcategory matching the parent-to-child relationship of categories. The nesting of categories can continue and return a large amount of data entirely dependent on the category hierarchy you have designed in your storefront.

To return data for only specific categories, you may want to use the GET method for a specific category.

Request

The URI call structure is:

GET http://{domain}/api/v1/categories?slug=slug

Parameters

The following parameter can be entered into the request:

Parameter Description Type
slug String providing the category with SEO optimized URL keywords. Example: fitness-shoes. Number

Response

The response includes all main and subcategories, displaying this content in a collection for every category located and returned from the storefront. Due to the parent-child relationship of categories, the data may return multiple nested subcategories with further nested subcategories providing a lengthy response.

Status Code 200

Data

The response includes a section of JSON detailing the pagination of the category content for displaying and navigating the information. Nested attribute data follows with detailed information per category and subcategory. Due to the parent-child relationship of categories, the data may return multiple nested subcategories with further nested subcategories.

The following data is returned as a collection of categories, one set of attributes per category with additional subcategories for the request:

Attribute Description Type
id The generated ID for the category when created. This is generated automatically by the Volusion system. Example: 1814 String
name The category name entered at category creation. The name is alphanumeric and supports some special characters. This is a required setting and always returns content. Example: Apparel. String
url The generated from the entered Category URL Text for the SEO options for a category. A dependency for this value is required through the Marketing > SEO > Search Engine Friendly URLs section of the Admin. With this activated and content entered, a new alternate, SEO friendly category displays. If NULL, the URL may default to the category ID.

String
imageUrl This text sets the URL path for the category image. If no image is included for the category, the setting defaults to a placeholder image URL. Example: /v/vspfiles/templates/99999/images/nophoto-0.gif String
description This option sets the description displayed for the category, on the category page when loaded in the storefront. The content can be deeply detailed, supporting plain text and HTML code. Sgtring
slug

This value provides the SEO friendly name for the category. A category may have a long name or contain numerous special characters. This content may not adhere to SEO usage. The name is used when generating the URL.

 String
subCategories

This is a collection of all subcategories to this category. Any category with the Subcategory of... entered with a parent category ID will be considered a subcategory of that parent category and display in that collection. The collection includes the following information:

  • id - The generated ID of the category.
  • name - The entered name for the category.
  • slug- Provides a SEO friendly name for the category. A category may have a long name or contain numerous special characters. This content may not adhere to SEO usage. The name is used when generating the URL.
  • url - Generated URL using SEO options and entered text to create an easily read and used category URL. Otherwise, it defaults to the category ID.
  • imgUrl - Sets the URL location of the category image to use and display. If no image is included for the category, the setting defaults to a placeholder image URL.
  • description - Fully defined description of the category.
  • subcategories - A collection of categories with this category listed as the parent category ID.
